# docs目录说明

## 目录作用

`docs/` 保存项目技术路线、系统分析报告、最终技术报告以及开发设计记录。该目录主要用于论文撰写、项目交付和后续维护。

## 文件说明

| 文件 | 作用 |
|---|---|
| `es_adaptive_sounding_final_technical_report.md` | 当前最完整的中文技术与实验分析报告，包含系统结构、原理、实验设计、结果评价和 SCI 投稿建议。 |
| `es_system_technical_analysis_report.md` | 系统技术分析报告，记录较早阶段的系统流程和测试结果。 |
| `es_task_driven_optimization_technical_route.md` | 任务驱动优化技术路线文档，说明任务模式、固定扫频范围、NSGA-II 和测试方案。 |
| `superpowers/` | 使用 Superpowers 技能过程生成的设计与计划文档。 |

## 推荐阅读顺序

1. `es_adaptive_sounding_final_technical_report.md`
2. `es_task_driven_optimization_technical_route.md`
3. `es_system_technical_analysis_report.md`
4. `superpowers/specs/` 与 `superpowers/plans/`

## 维护说明

- 新的论文级总结优先写入 `es_adaptive_sounding_final_technical_report.md`。
- 如果代码或实验结果发生大改，应同步更新该目录下的报告。
